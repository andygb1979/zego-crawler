# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x: |

## Reporting a Vulnerability

If you find a security issue, please open a private report via GitHub **Security → Advisories → Report a vulnerability** on this repository, or contact the maintainer directly if you already have their details.

Include:

- A description of the issue and potential impact
- Steps to reproduce, or a proof of concept if available
- The version or commit you tested against

You should receive an acknowledgement within a few business days. If the report is accepted, we will work on a fix and coordinate disclosure; if declined, we will explain why.

Please do not open public GitHub issues for undisclosed security vulnerabilities.
